recipe-publisher — pipeline

Structure, end-to-end flow with a worked example, and future ideas. Internal doc.

1. Overview

Turns a single seed (a structured recipe brief) into a published recipe across WordPress, Instagram (carousel), Facebook, and Pinterest — Nalla's Dad voice, one operator gate (audio + verify) between offline prep and the daily publish cron, idempotent on retry.

Entry points (two-phase, seed-driven): prepare.py --seed <id> builds all artifacts into campaigns/prepared/<id>/; scripts/publish_prepared.py (daily 14:00 cron) drains the queue once status flips to verified. Adjacent: recipe_publisher.py (legacy topic-based, free-text) and scripts/content_pipeline.py --stage {reel,campaign,ideate,publish}. ideator/ generates new seeds upstream.

3. Flow — worked example

Seed: berry-yogurt-swirl-frozen-pupsicles. Two phases: prepare (offline build) → verify (operator gate) → publish (cron).

1. Prepare
python prepare.py --seed berry-yogurt-swirl-frozen-pupsicles. Resolves seed from seeds.json + carousel spec from seeds/carousels/{slug}.json.
2. Draft
generators/recipe.py calls the active Drafter (Gemini if VOICE_PROVIDER=gemini, else Anthropic). Produces recipe body + IG/FB captions. _validate() hard-rejects missing CTA / bullets / branded hashtag.
3. Images
generators/carousel.py renders 4 slides via Imagen / Nano Pro. Slide 1 stamped with @dogfoodandfun follow badge; slide 4 stamped with FULL RECIPE → DOGFOODANDFUN.COM ribbon. Featured.jpg written for WP hero.
4. WP draft
publishers/wordpress.py uploads featured image, sets fifu_image_url + fifu_image_alt meta, creates a draft post with clean HTML (Elementor edit meta stripped). wp_draft_id persisted in metadata.json.
5. Persist
All artifacts land in campaigns/prepared/<seed>/. status.json = awaiting_audio.
6. Audio
manual Operator generates the music bed in Suno (or similar), drops audio.mp3 into the prepared folder. Pipeline does not author music or lyrics for recipes — Jamendo is reel-only. Status flips to audio_ready.
7. Verify
python scripts/publish_prepared.py --seed <id> --verify sends a Telegram preview (body + slides + captions + audio). On approve, status flips to verified.
8. Publish
Daily 14:00 cron publish_prepared.py picks the next verified campaign and runs publishers in order: WP draft → publish → IG carousel (container → poll ≤ 5min → publish) → FB page post (permalink inline, never "link in first comment") → Pinterest pin board blocked: Trial-tier API. Idempotent on crash via per-step status.json.
9. Archive
Folder moves to campaigns/published/<seed>/. publishing_timeline.json updated with platform permalinks + pin IDs. Daily report written under /mnt/dogfoodandfun/.

5. Future ideas

  • growth Auto-warmup scheduling. Newly-joined FB groups currently enforce 48h-comment / 72h-link cooldown by check. Promote that to a real scheduler so warmup actions get queued, not just gated.
  • content Reel as a publisher target. Today reels are a separate --stage reel entrypoint. Fold them into the main recipe_publisher.py so a single seed run can emit carousel + reel + WP post in one approval cycle.
  • content Per-seed lyrics drafter. generators/lyrics_drafter.py is scaffolded — wire it into the reel pipeline so each recipe gets a custom 4-line jingle instead of generic Jamendo backing.
  • revenue Recipe-product attachments. lib/recipe_products/ + data/recipe_products.json already model "this recipe pairs with X tool / Y ingredient." Surface them as affiliate boxes in the WP post + last-slide CTA on the carousel.
  • distribution Pinterest API uplift. Apply for Standard access to unblock the auto-pin step. Memory: blocked on Pinterest Trial tier 403 (code 29). Worth chasing — Pinterest is the highest-leverage long-tail driver for recipe content.
  • distribution TikTok publisher. OAuth redirect URI is already set up (/tiktok/callback WP page). Add a publishers/tiktok.py that takes the existing 9:16 reel mp4 and publishes via the Content Posting API.
  • quality Draft-history dedup across IG + WP. lib/draft_history.py exists but isn't yet consulted by every drafter — wire it in so we never re-pitch the same hook/headline within a rolling 60-day window.
  • ops One report file per surface. Today the daily report bundles every platform's outcome. Split into per-platform tabs so failures are visible without scrolling — easier on-call.
  • ops Replace JWT REST paywall. The wp-rest-api-authentication plugin (miniorange free tier) blocks every 3rd-party REST namespace including Code Snippets. Either upgrade or swap for a free alternative — would unlock fully-automated footer/snippet edits.
