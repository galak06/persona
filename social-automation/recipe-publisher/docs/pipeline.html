recipe-publisher — pipeline

Structure, end-to-end flow with a worked example, and future ideas. Internal doc.

1. Overview

Turns a single seed (a structured recipe brief) into a published recipe across WordPress, Instagram (carousel), Facebook, and Pinterest — Nalla's Dad voice, Telegram approval gates between stages, fully idempotent.

Entry points: recipe_publisher.py (full pipeline) and scripts/content_pipeline.py --stage {reel,campaign} (specialized variants). The ideator/ module generates new seeds upstream.

3. Flow — worked example

Seed: berry-yogurt-swirl-frozen-pupsicles (new this week). Command: python recipe_publisher.py --topic berry-yogurt-swirl-frozen-pupsicles --no-dry-run

1. Load
load_config() hydrates env from .claude/settings.local.json; load_state() reads state/published_recipes.json to dedup.
2. Resolve seed
seeds.json → {slug, title, ingredients, prep_minutes, ...}. Paired carousel spec loaded from seeds/carousels/berry-yogurt-swirl-frozen-pupsicles.json.
3. Draft
generators/recipe.py calls the active Drafter (Gemini if VOICE_PROVIDER=gemini, else Anthropic). Produces recipe body + IG caption. _validate() hard-rejects missing CTA / bullets / branded hashtag.
4. Images
generators/carousel.py renders 4 slides via Imagen/Nano Pro. Slide 1 stamped with @dogfoodandfun follow badge; slide 4 stamped with FULL RECIPE → DOGFOODANDFUN.COM ribbon.
5. Approve
Telegram preview (markdown body + 4 slides) → approve / skip via inline buttons. Polls up to 12h, then aborts.
6. WordPress
publishers/wordpress.py: upload hero to media, set fifu_image_url + fifu_image_alt meta, strip Elementor edit meta so the clean HTML wins, publish post.
7. Instagram
publishers/instagram.py::publish_carousel: upload each slide, create media_type=CAROUSEL children, parent container, poll status ≤ 5 min, publish.
8. Facebook
publishers/facebook.py: page post with WP permalink inline (link in body, never "link in first comment" — that's page-algo-only).
9. Pinterest
publishers/pinterest.py: pin slide 1 to the recipe board with WP permalink as destination. blocked: Trial-tier API
10. Persist
Append entry to state/published_recipes.json, update state/publishing_timeline.json, write daily report to /mnt/dogfoodandfun/recipe-publisher-report-YYYY-MM-DD.md.

5. Future ideas

  • growth Auto-warmup scheduling. Newly-joined FB groups currently enforce 48h-comment / 72h-link cooldown by check. Promote that to a real scheduler so warmup actions get queued, not just gated.
  • content Reel as a publisher target. Today reels are a separate --stage reel entrypoint. Fold them into the main recipe_publisher.py so a single seed run can emit carousel + reel + WP post in one approval cycle.
  • content Per-seed lyrics drafter. generators/lyrics_drafter.py is scaffolded — wire it into the reel pipeline so each recipe gets a custom 4-line jingle instead of generic Jamendo backing.
  • revenue Recipe-product attachments. lib/recipe_products/ + data/recipe_products.json already model "this recipe pairs with X tool / Y ingredient." Surface them as affiliate boxes in the WP post + last-slide CTA on the carousel.
  • distribution Pinterest API uplift. Apply for Standard access to unblock the auto-pin step. Memory: blocked on Pinterest Trial tier 403 (code 29). Worth chasing — Pinterest is the highest-leverage long-tail driver for recipe content.
  • distribution TikTok publisher. OAuth redirect URI is already set up (/tiktok/callback WP page). Add a publishers/tiktok.py that takes the existing 9:16 reel mp4 and publishes via the Content Posting API.
  • quality Draft-history dedup across IG + WP. lib/draft_history.py exists but isn't yet consulted by every drafter — wire it in so we never re-pitch the same hook/headline within a rolling 60-day window.
  • ops One report file per surface. Today the daily report bundles every platform's outcome. Split into per-platform tabs so failures are visible without scrolling — easier on-call.
  • ops Replace JWT REST paywall. The wp-rest-api-authentication plugin (miniorange free tier) blocks every 3rd-party REST namespace including Code Snippets. Either upgrade or swap for a free alternative — would unlock fully-automated footer/snippet edits.
